# Semantic Sync + Metric Calc Console - Quick Reference

## 🎯 What's New

| Component | Location | Purpose |
|-----------|----------|---------|
| **Semantic Sync** | `services/semantic-sync/main.go` | Listens to Postgres, generates Cube.js schemas in real-time |
| **Metric Console** | `frontend/src/pages/metrics/MetricCalcConsole.tsx` | React UI for metric registry, PoP trends, anomalies |
| **Trigger Migration** | `db/migrations/20251104_...sql` | Postgres trigger for metric_registry change notifications |
| **Docker Service** | `docker-compose.yml` | Added `semantic-sync` service |
| **Navigation** | `MainNavigation.tsx` | Added "Metric Calc" to Entity → Entities menu |
| **Routing** | `AppRoutes.tsx` | Added `/metrics/calc-console` route |

---

## 🚀 Start in 3 Steps

```bash
# Step 1: Create Postgres trigger
psql postgres://postgres:postgres@host.docker.internal:5432/alpha -f \
  db/migrations/20251104_add_metric_registry_notify_trigger.sql

# Step 2: Start services (includes semantic-sync)
docker-compose up -d

# Step 3: Open console
open http://localhost:3000/metrics/calc-console
```

## 🔌 Architecture

```
React Console (UI)
    ↓ REST API
Backend (Temporal + RabbitMQ)
    ↓ SQL
Postgres (metric_registry)
    ↓ NOTIFY
Semantic Sync Service (Go)
    ↓ Write
Cube Schemas (./cube-schemas/)
```

---

## 📊 Cube Schemas Generated

**Automatically created in `./cube-schemas/`**:

1. **metrics_pop.js** - Period-over-period trends
2. **metrics_anomalies.js** - Anomaly detection results
3. **metrics_atomic.js** - Base atomic metrics

✅ Auto-updates when metrics change

## 🔍 Verify It's Working

```bash
# 1. Check semantic-sync is running
docker ps | grep semantic-sync

# 2. Check trigger created
psql -d alpha -c "SELECT tgname FROM pg_trigger WHERE tgname LIKE 'metric_registry%';"

# 3. Check schemas exist
ls -la cube-schemas/

# 4. Manually test notification
psql -d alpha -c "INSERT INTO metric_registry 
  (metric_id, tenant_id, name, domain) 
  VALUES ('test-1', 'tenant-1', 'Test', 'Finance');"

# 5. Check schema regenerated
ls -lat cube-schemas/ | head -1
```

## ⚠️ Troubleshooting

| Issue | Solution |
|-------|----------|
| Semantic sync won't start | Check DATABASE_URL, postgres running |
| Schemas not updating | Run trigger migration, restart semantic-sync |
| Console won't load | Verify route in AppRoutes.tsx, clear cache |
| No metrics showing | Create test metric, check backend API |
