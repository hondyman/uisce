💡 KEY CONCEPTS

Business Object (BO)
  Defines what data to capture:
  • Fields (employee_id, first_name, etc.)
  • Validation rules (format, length, etc.)
  • Relationships (department, position)

UI Layout
  Defines how to present the BO:
  • Sections (Basic, Contact, Employment)
  • Columns (1, 2, or 3)
  • Actions (Save, Submit, Cancel)

Validation Engine
  Validates form data:
  • Rules are reusable
  • Multiple rules per field
  • Error/warning severity

Multi-Tenant Scoping
  Isolates data by tenant:
  • X-Tenant-ID header
  • X-Tenant-Datasource-ID header
  • Enforced at request, auth, and DB level
