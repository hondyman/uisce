🎯 KEY RECOMMENDATIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. **Start with Code Templates**
   Don't write from scratch. PHASE_6B_STARTER_CODE.md has 80% of code ready.
   Just customize for your needs.

2. **Use Native Drag API (Not react-flow)**
   For simplicity, use native HTML5 drag API. Easier to understand.
   react-flow adds complexity not needed for MVP.

3. **Test Locally with Curl First**
   Before building UI, verify API works:
   curl -X POST http://localhost:8080/api/bp ...
   This ensures integration points are solid.

4. **Parallel Task Work**
   While building UI (Task 4), start writing tests (Task 6).
   Tests clarify expected behavior. Speeds up development.

5. **Daily Demos**
   Show progress to team every day. Get feedback early.
   Iterate quickly. Ship fast.

6. **Use git Branches**
   Task 4: feature/bp-ui-builder
   Task 5: feature/bp-demo
   Task 6: feature/bp-tests
   Merge when each is complete.


⚠️ COMMON PITFALLS TO AVOID
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

❌ Forgetting tenant_id scoping
   └─ Always check: WHERE tenant_id = $1 AND datasource_id = $2

❌ Blocking approval workflow
   └─ Use async/await for long operations. Show loading spinner.

❌ Not testing timeout logic
   └─ Explicitly test: workflow exceeds deadline → timeout fires

❌ Incomplete error handling
   └─ Catch DB errors, network errors, validation errors.
   └─ Display user-friendly messages.

❌ Over-complicating drag-drop
   └─ Native HTML5 drag API is sufficient. Don't add react-flow unless needed.

❌ Skipping integration tests
   └─ End-to-end tests catch bugs missed by unit tests.
   └─ Test the HireEmployee workflow fully.
