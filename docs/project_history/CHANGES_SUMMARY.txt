BEHAVIOR CHANGES
================================================================================

BEFORE:
  POST /api/metrics/{id}/compute/pop
  ├─ Create job_run (status: pending)
  ├─ Return 202 (job_run never updated to success)
  └─ No asynchronous execution

AFTER:
  POST /api/metrics/{id}/compute/pop
  ├─ Create job_run (status: pending)
  ├─ Trigger Temporal workflow (asynchronous)
  │  └─ Workflow updates: pending → running → success/failed
  ├─ Publish RabbitMQ event on completion
  └─ Return 202 immediately

================================================================================
QUICK START
================================================================================

1. Start Temporal:    temporal server start-dev
2. Start RabbitMQ:    docker run -d -p 5672:5672 -p 15672:15672 rabbitmq:4.0-management
3. Set environment:   export TEMPORAL_HOSTPORT=localhost:7233 RABBIT_URL=amqp://guest:guest@localhost:5672/
4. Initialize DB:     psql < backend/sql/calc-engine.sql
5. Start backend:     cd backend && go run ./cmd/server/main.go
6. Test:              bash test-calc-engine.sh
